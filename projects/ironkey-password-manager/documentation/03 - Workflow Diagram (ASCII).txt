=== IronKey KeePassXC Portable – Workflow Diagram (ASCII) ===

This diagram illustrates the complete operational workflow for the
IronKey portable KeePassXC system across Windows and Linux, including
unlocking, execution, provenance verification, and zero‑residue cleanup.

----------------------------------------------------------------------
1. High-Level System Overview
----------------------------------------------------------------------

                   +-----------------------------+
                   |     IronKey D500S (FAT32)   |
                   |-----------------------------|
                   | KeePassXC-Linux/            |
                   | KeePassXC-Windows/          |
                   | IronKeyVault/               |
                   | ironkey-keepass.sh          |
                   | SOURCE + DIGEST files       |
                   +-----------------------------+
                                |
                                v
                   +-----------------------------+
                   |   Host System (Win/Linux)   |
                   +-----------------------------+

----------------------------------------------------------------------
2. Provenance Verification Flow
----------------------------------------------------------------------

                +----------------------+
                |  SOURCE.txt          |
                |  DIGEST (SHA-256)    |
                +----------+-----------+
                           |
                           v
                +----------------------+
                |  User runs sha256sum |
                +----------+-----------+
                           |
                           v
                +----------------------+
                |  Hash matches?       |
                +----------+-----------+
                           |
          +----------------+----------------+
          |                                 |
         Yes                               No
          |                                 |
          v                                 v
+----------------------+        +-----------------------------+
|  Execute workflow    |        |  STOP: Do not run binary    |
+----------------------+        +-----------------------------+

----------------------------------------------------------------------
3. Windows Workflow
----------------------------------------------------------------------

+---------------------------+
| 1. Unlock IronKey         |
+-------------+-------------+
              |
              v
+---------------------------+
| 2. Navigate to            |
|    KeePassXC-Windows/     |
+-------------+-------------+
              |
              v
+---------------------------+
| 3. Run KeePassXC.exe      |
+-------------+-------------+
              |
              v
+---------------------------+
| 4. Open IronKeyVault.kdbx |
+-------------+-------------+
              |
              v
+---------------------------+
| 5. Close KeePassXC        |
|    (zero residue)         |
+---------------------------+

----------------------------------------------------------------------
4. Linux Workflow
----------------------------------------------------------------------

+---------------------------+
| 1. Unlock IronKey         |
+-------------+-------------+
              |
              v
+---------------------------+
| 2. Copy runner script     |
|    to native FS           |
|    (ext4/xfs/btrfs)       |
+-------------+-------------+
              |
              v
+---------------------------+
| 3. chmod +x script        |
+-------------+-------------+
              |
              v
+---------------------------+
| 4. Run script             |
+-------------+-------------+
              |
              v
+---------------------------+
| Script actions:           |
|  - Copy AppImage → /tmp   |
|  - chmod +x AppImage      |
|  - Launch KeePassXC       |
|  - Wait for exit          |
|  - Delete AppImage        |
|  - Delete itself          |
|  (No installation occurs) |
+-------------+-------------+
              |
              v
+---------------------------+
| 5. Zero residue achieved  |
+---------------------------+

----------------------------------------------------------------------
5. Data Flow Summary
----------------------------------------------------------------------

Windows:
    IronKey → KeePassXC.exe → Vault → Close → Zero residue

Linux:
    IronKey → Copy script → Native FS → /tmp/AppImage → Execute → Cleanup

Provenance:
    SOURCE.txt + DIGEST → sha256sum → Verified → Execute

----------------------------------------------------------------------
6. Zero-Residue Lifecycle
----------------------------------------------------------------------

+---------------------------+
| 1. No installation        |
+---------------------------+
| 2. No registry writes     |
+---------------------------+
| 3. No host config files   |
+---------------------------+
| 4. No temp files remain   |
+---------------------------+
| 5. Script self-deletes    |
+---------------------------+
| 6. AppImage deleted       |
+---------------------------+
| 7. Vault stays on IronKey |
+---------------------------+

----------------------------------------------------------------------
End of Workflow Diagram

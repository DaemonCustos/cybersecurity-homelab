=== IronKey KeePassXC Portable – Consolidated Technical Documentation ===

This document summarizes the validated architecture, the functional
workflow, the failure points encountered during development, and the
technical reasons behind each design decision. All information reflects
the final, working state of the IronKey portable KeePassXC solution.

------------------------------------------------------------
1. Final Architecture Overview
------------------------------------------------------------
- IronKey D500S secure partition (FAT32)
- KeePassXC AppImage for Linux (portable, no installation required)
- KeePassXC.exe for Windows (portable)
- KeePassXC database stored on IronKey
- Linux runner script stored on IronKey but executed from native FS
- AppImage executed from /tmp due to FAT32 + showexec limitations
- Zero-residue execution on both Linux and Windows

------------------------------------------------------------
2. Linux Execution Model
------------------------------------------------------------
Linux cannot execute binaries or scripts directly from the IronKey
secure partition. The secure partition is FAT32 and is auto-mounted
by udisks2 with the "showexec" flag. This prevents execution of:

- AppImages
- ELF binaries
- Shell scripts
- Any file without a Windows executable extension

This is a filesystem limitation, not an installation requirement.

The AppImage remains portable. It simply must be executed from a native
filesystem that supports Unix permissions.

The validated Linux workflow is:

1. Copy the runner script from IronKey to a native filesystem.
2. Execute the script from the native filesystem.
3. Script copies the AppImage to /tmp.
4. Script marks the AppImage executable.
5. Script runs KeePassXC directly from /tmp.
6. Script deletes the AppImage.
7. Script deletes itself.

This ensures zero residue on the Linux host.

------------------------------------------------------------
3. Windows Execution Model
------------------------------------------------------------
Windows can execute KeePassXC.exe directly from the IronKey. No script
is required. KeePassXC portable does not write registry entries or
persistent configuration files. No residue remains after closing the
application.

------------------------------------------------------------
4. Password Mode Requirement
------------------------------------------------------------
The IronKey D500S supports two password modes:

- Passphrase Mode
- Complex Mode

Linux supports ONLY Complex Mode.  
Passphrase Mode does not work on Linux due to a firmware limitation in
the IronKey Linux unlocker.

Windows supports both modes.

For cross-platform use, the IronKey must be configured in Complex Mode.

------------------------------------------------------------
5. What Failed and Why
------------------------------------------------------------
A. Executing scripts or AppImages directly from IronKey
- Failed due to FAT32 + "showexec" mount behavior.
- Linux treats all non-Windows extensions as non-executable.
- chmod +x has no effect on FAT32.
- udisks2 overrides remount attempts.

B. Remounting with exec
- Command succeeded syntactically but had no effect.
- udisks2 re-applied "showexec" immediately.

C. Running Windows-authored scripts on Linux
- Failed due to CRLF line endings.
- Linux requires LF-only for shebang recognition.

D. Attempting to bypass FAT32 restrictions
- Not possible without modifying system-level udisks2 policies.
- Not acceptable for a portable, host-agnostic workflow.

------------------------------------------------------------
6. What Worked and Why
------------------------------------------------------------
A. Running the AppImage from /tmp
- Native Linux filesystems support executable permissions.
- The AppImage runs directly and portably from /tmp without installation.

B. Self-scrubbing runner script
- Script executes from native FS.
- Script performs permanent deletion (not Trash).
- Leaves zero residue.

C. Windows portable execution
- KeePassXC.exe runs directly from IronKey.
- No installation or residue.

D. Creating the script directly on Linux
- Avoids CRLF contamination.
- Ensures correct permissions and shebang behavior.

------------------------------------------------------------
7. Final Validated Workflow
------------------------------------------------------------
Linux:
1. Unlock IronKey.
2. Copy runner script to native filesystem.
3. chmod +x the script.
4. Run the script.
5. Script copies AppImage to /tmp.
6. Script runs KeePassXC.
7. Script deletes AppImage and deletes itself.

Windows:
1. Unlock IronKey.
2. Navigate to KeePassXC-Windows.
3. Run KeePassXC.exe directly.
4. Close KeePassXC; no residue remains.
